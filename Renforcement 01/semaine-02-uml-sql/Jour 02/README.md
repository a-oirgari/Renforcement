# Jour 02 - UML : Diagrammes de cas d'utilisation

Vous devez completer les fichiers `.puml` en suivant les specifications dans les fichiers `.md`.
Utilisez Draw.io ou StarUML si vous preferez une interface graphique.

Les relations a maitriser :
- `actor -> usecase` : l'acteur declenche le cas d'utilisation
- `<<include>>` : comportement obligatoirement inclus (factorisation)
- `<<extend>>` : comportement optionnel qui etend un cas de base
- `usecase A --> usecase B : <<include>>`

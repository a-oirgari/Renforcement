# Jour 03 - UML : Diagrammes de classes

Vous concevez les diagrammes de classes et les transformez en schema de base de donnees.

Rappels sur les relations :
- Association : une classe connait une autre (fleche simple)
- Composition : partie ne peut exister sans le tout (losange plein)
- Agregation : partie peut exister sans le tout (losange vide)
- Heritage : is-a (fleche triangle)
- Cardinalites : 1, 0..1, 0..*, 1..*

Pour chaque exercice :
1. Realiser le diagramme de classes
2. Deduire le schema relationnel (CREATE TABLE)
3. Identifier les index et contraintes d'integrite
